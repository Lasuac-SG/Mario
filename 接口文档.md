下面是按「**View↔ViewModel↔Model，两端都只经 ViewModel 中转**」原则整理的完整接口文档。它基于现在已写好的函数，并标注了**为满足该原则需要做的重构改动**（主要是切断 View 直接访问 Model 的两条现存违例：`vm_->gameModel().xxx()` 和 `EntityRenderer` 直接吃 `const GameModel&`）。

# Mario MVVM 接口文档

## 0. 架构原则与依赖方向

```
   输入(下行)                        通知+数据(上行)
View ──inputCmd/tick──▶ ViewModel ──setMove*/update──▶ Model
View ◀──vmTrigger/DTO── ViewModel ◀──modelTrigger/getter── Model
```

- **依赖方向**：`View → ViewModel → Model → common`（单向）。View **不 include** `model/*`；Model **不 include** `view/*`、`viewmodel/*`。
- **common 是共享底座**：三层都可用其枚举与工具类（`MarioState/TileType/Direction/EventTrigger/ICommand`）。
- **ViewModel 是唯一中转**：向下调用 Model、订阅 Model 通知；向上给 View 提供命令、数据快照（已换算为相对/屏幕坐标）和重绘事件。

---

## 1. common 层（共享基础设施，无跨层调用）

| 文件 | 符号 | 作用 |
|---|---|---|
| `EventId.h` | `enum MarioState{IDLE,RUNNING,JUMPING,FALLING}` | 玩家动画状态 |
| | `enum TileType{EMPTY,GROUND,PIPE,PLATFORM}` | 瓦片类型 |
| | `enum Direction{LEFT,RIGHT}` | 朝向 |
| | `enum ModelEvent{STATE_CHANGED}` | Model→VM 事件 id |
| | `enum ViewModelEvent{RENDER_UPDATE}` | VM→View 事件 id |
| | `enum InputAction{MOVE_LEFT,MOVE_RIGHT,STOP,JUMP,RESTART}` | 输入语义 |
| `Type.h` | `PositionType=float`；`EventType=uint32_t` | 类型别名 |
| | `TypeParameter<T>{ explicit TypeParameter(const T&); T v_; }` | 命令参数包装 |
| `ICommand.h` | `ICommandBase::exec(ICommandParameter*)->int` | 命令接口 |
| `EventTrigger.h/.cpp` | `add_notification(Notify_Funtion&&)->uintptr_t` | 注册回调，返句柄 |
| | `remove_notification(uintptr_t)` | 退订 |
| | `fire(EventType)` | 广播事件 id |

> 说明：通知机制**只传事件 id，不带数据**；数据由收到通知方回头拉取。

---

## 2. model 层（只与 ViewModel 通信）

**职责**：纯游戏逻辑（像素坐标物理 + 瓦片碰撞），对外提供输入入口、驱动、数据 getter、变更通知。**不认识 ViewModel/View**——只通过 `modelTrigger.fire` 被动地被订阅。

### `GameModel`（`model/GameModel.h/.cpp`）—— Model 对外唯一门面
| 函数                  | 签名                                              | 功能                                | 供谁调             |
| ------------------- | ----------------------------------------------- | --------------------------------- | --------------- |
| `update`            | `void update(TimeType dt)`                      | 按子步长推进物理，末尾 `notifyChanged()`     | ViewModel::tick |
| `setMoveLeft/Right` | `void setMoveLeft(bool)` / `setMoveRight(bool)` | 设左/右移动意图                          | InputCommand    |
| `setMoveStop`       | `void setMoveStop()`                            | 清移动意图                             | InputCommand    |
| `jump`              | `void jump()`                                   | 着地时起跳                             | InputCommand    |
| `reset`             | `void reset()`                                  | 重载关卡+归位+重建瓦片+通知                   | InputCommand    |
| `loadLevelFromFile` | `bool loadLevelFromFile(const std::string&)`    | 换关卡（可选）                           | ViewModel(可选)   |
| `playerX/Y/W/H`     | `PositionType ...() const`                      | 玩家**绝对像素**位置/尺寸                   | ViewModel       |
| `playerState`       | `MarioState playerState() const`                | 玩家动画状态                            | ViewModel       |
| `playerFacing`      | `Direction playerFacing() const`                | 玩家朝向                              | ViewModel       |
| `tiles`             | `const std::vector<Tile>& tiles() const`        | 实心瓦片列表（**绝对像素**，含 `x,y,w,h,type`） | ViewModel       |
| `modelTrigger`      | `EventTrigger`（public 成员）                       | 状态变化时 `fire(STATE_CHANGED)`       | ViewModel 订阅    |

**内部调用**（不跨层）：`update`→`Mario::step`；`reset/load`→`TileMap::loadFrom*`、`Mario::reset`、`rebuildTiles`；`rebuildTiles`→`TileMap::at/cols/rows`。

**Model 调用的其它层函数**：仅 `common` 的 `EventTrigger::fire`（在 `notifyChanged`）。**不调用 ViewModel/View。**

（辅助类 `Mario`、`TileMap`、`PhysicsConfig`、`Tile` 为 Model 内部实现，不对上层暴露。）

---

## 3. viewmodel 层（中转：向下驱动 Model，向上服务 View）

**职责**：持有 `GameModel*`；把 View 的输入翻译成 Model 调用；订阅 Model 通知并转成 View 的重绘事件；把 Model 的**绝对坐标**数据换算成**相对/屏幕坐标 DTO** 提供给 View。

### 需新增的 View 面向 DTO（建议放 `viewmodel/ViewData.h`，供 VM+View 共用）
```cpp
struct PlayerView { float x, y, w, h; MarioState state; Direction facing; }; // 相对坐标
struct TileView   { float x, y, w, h; TileType type; };                      // 相对坐标
```

### `GameViewModel`（`viewmodel/ViewModel.h/.cpp`）
| 函数 | 签名 | 功能 | 内部调用（取什么数据） |
|---|---|---|---|
| 构造 | `GameViewModel(GameModel*)` | 保存指针；`modelTrigger.add_notification(onModelChanged)` 订阅 Model | 调 `GameModel::modelTrigger.add_notification` |
| 析构 | `~GameViewModel()` | `modelTrigger.remove_notification(subId)` | 调 `GameModel::modelTrigger.remove_notification` |
| `tick` | `void tick(float dt)` | 驱动一帧 | 调 `GameModel::update(dt)` |
| `inputCmd` | `ICommandBase& inputCmd()` | 返回输入命令给 View | —（View 拿去 exec） |
| `playerView` | `PlayerView playerView() const` | **【新增】** 组装玩家 DTO，绝对→相对换算 | 调 `GameModel::playerX/Y/W/H/State/Facing`，取玩家状态 |
| `tiles` | `std::vector<TileView> tiles() const` | **【新增】** 组装瓦片 DTO，绝对→相对换算 | 调 `GameModel::tiles()`，取瓦片列表 |
| `levelSize` | `sf::Vector2f levelSize() const`（或 `float w()/h()`） | **【新增】** 关卡尺寸，供 View 布局/相机 | 需 Model 提供关卡尺寸 getter（见重构清单） |
| `vmTrigger` | `EventTrigger`（public 成员） | 变更时 `fire(RENDER_UPDATE)` | View 订阅 |
| `onModelChanged` | `void onModelChanged(EventType)`（private） | 收到 `STATE_CHANGED` → 更新相机 → `vmTrigger.fire(RENDER_UPDATE)` | 被 `GameModel::modelTrigger` 回调触发 |

### `InputCommand`（`viewmodel/GameCommands.h`）
| 函数 | 签名 | 功能 | 内部调用 |
|---|---|---|---|
| `exec` | `int exec(ICommandParameter*)` | 解析 `InputActionParameter`，按 `InputAction` 分派 | 调 `GameModel::setMoveLeft/Right/setMoveStop/jump/reset` |

**ViewModel 调用的其它层函数**：向下全是 `GameModel::*`（驱动+取数）；向上不主动调 View，只经 `vmTrigger` 被订阅。

## 4. view 层（只与 ViewModel 通信）

**职责**：建窗口、收键盘、每帧驱动 VM、拉 VM 的 DTO 渲染。**不再 include/调用 `GameModel`。**

### `GameView`（`view/View.h/.cpp`）
| 函数 | 签名 | 功能 | 调用 ViewModel 的什么（取什么数据） |
|---|---|---|---|
| 构造 | `GameView(GameViewModel*, EntityRenderer*)` | 建窗口/持有 VM 与渲染器；可订阅 `vm->vmTrigger` | — |
| `run` | `void run()` | 主循环：事件→输入→`tick`→渲染 | 调 `vm->tick(dt)` |
| `processInput` | `void processInput()`（private） | 按键→`InputActionParameter`→exec | 调 `vm->inputCmd().exec(&p)`（下达输入） |
| `render` | `void render()`（private） | 画瓦片与玩家 | 调 `vm->tiles()` 取瓦片 DTO、`vm->playerView()` 取玩家 DTO |

### `EntityRenderer` / `SimpleRenderer`（`view/EntityRenderer.h`,`SimpleRenderer.*`）
| 函数 | **重构后**签名 | 功能 |
|---|---|---|
| `drawPlayer` | `void drawPlayer(sf::RenderWindow&, const PlayerView&, AssetManager&)` | 按状态/朝向选帧并绘制（**改吃 DTO，不再吃 `const GameModel&`**） |
| `drawTile` | `void drawTile(sf::RenderWindow&, const TileView&, AssetManager&)` | 按 `type` 绘制（**改吃 `TileView`，不再吃 model 的 `Tile`**） |

### `AssetManager`（`view/AssetManager.h`）
| 函数 | 签名 | 功能 |
|---|---|---|
| `load` | `const sf::Texture& load(const std::string&)` | 加载并缓存纹理 |

**View 调用的其它层函数**：全部是 `GameViewModel::*` 与 `common` 枚举/命令；**零 `GameModel` 调用**。

## 5. 完整数据流（按调用链）

**A. 输入下行（改马里奥意图）**
```
键盘 → GameView::processInput
     → GameViewModel::inputCmd()                 [ViewModel.h]
     → InputCommand::exec(InputActionParameter)  [GameCommands.h]
     → GameModel::setMoveLeft/Right/Stop/jump/reset [GameModel.h]
     → Mario::setMoveLeft/... （只置意图，不发通知） [Mario.h]
```

**B. 驱动 + 上行通知（Model→VM→View）**
```
GameView::run → GameViewModel::tick(dt)          [ViewModel.cpp]
             → GameModel::update(dt) → Mario::step→碰撞 → notifyChanged()
             → GameModel.modelTrigger.fire(STATE_CHANGED)  [EventTrigger.cpp]
             → (回调) GameViewModel::onModelChanged        [ViewModel.cpp]
             → GameViewModel.vmTrigger.fire(RENDER_UPDATE)
             → GameView 重绘
```

**C. 渲染取数（View 只问 VM）**
```
GameView::render
     → GameViewModel::tiles()      → 内部调 GameModel::tiles()（绝对坐标）→ 换算相对 → vector<TileView>
     → GameViewModel::playerView() → 内部调 GameModel::playerX/Y/W/H/State/Facing → 换算相对 → PlayerView
     → SimpleRenderer::drawTile / drawPlayer（吃 DTO）
```

---

## 6. 为满足原则需要的重构改动清单（相对现状）

1. **删除/私有化** `GameViewModel::gameModel()`——这是 View 直连 Model 的入口，必须切断。
2. **ViewModel 新增** `playerView()`、`tiles()`（返回 `PlayerView`/`vector<TileView>`，在此做**绝对→相对**坐标换算），以及关卡尺寸/相机所需的 getter。
3. **新增** `viewmodel/ViewData.h` 定义 `PlayerView`/`TileView`（View 与 VM 共用，不依赖 model）。
4. **`EntityRenderer`/`SimpleRenderer` 改签名**：`drawPlayer(...,const PlayerView&,...)`、`drawTile(...,const TileView&,...)`，去掉对 `const GameModel&` 和 model `Tile` 的依赖。
5. **`GameView::render` 改为**从 `vm_->tiles()/playerView()` 取数，删掉 `vm_->gameModel().xxx()`。
6. **Model 侧新增关卡尺寸 getter**（如 `levelWidthPx()/levelHeightPx()`）供 VM 算相机（这是我之前 `LevelView` 里有、后来精简掉的信息，按需恢复）。
7. **View 移除** `#include "model/GameModel.h"`（含 `EntityRenderer.h` 里的该 include），改 include `viewmodel/ViewData.h`。

完成后：**View 只依赖 viewmodel + common，Model 只依赖 common**，两端严格经 ViewModel 中转，符合你定的原则。
